# 咩了个咩 · 开发计划

> 把整个项目切成 7 个阶段。
> 每阶段都有「这阶段做完长啥样」的可见标志，做不到不进下一关。

---

## 心法

- **每阶段都要能玩 / 能看**：不允许"写了一周还没法运行"。
- **跑通 > 完美**：先丑后美、先少后多。
- **每阶段结束都 Git 提交**：打个 tag，万一翻车能退。
- **每阶段最后更新 `progress.md`**：写清这阶段做了啥、下阶段做啥。

---

## 阶段 0 · 准备工作（预计 0.5 ~ 1 天）

**目标**：环境搭好，能在本地打开一个空白 `index.html`，并且能在 GitHub 上看到。

**做完的标志**：

- [ ] 装好 VS Code（如果还没装）。
- [ ] 装好 Git。
- [ ] 注册 GitHub 账号。
- [ ] 在 GitHub 上建空仓库（建议英文名 `sheep-stack` 或 `meh-le-ge-meh`，URL 友好）。
- [ ] 把仓库 clone 到本地 `D:\羊了个羊\`。
- [ ] 写好 `README.md`、`.gitignore`、`LICENSE`。
- [ ] 写一个 `index.html`，里面只有 "Hello World"，能在浏览器打开。
- [ ] 第一次 `git commit` + `git push`。
- [ ] 在 GitHub 仓库设置里**开启 Pages**，确认能在线访问到 Hello World。

**这阶段我（Cascade）要先教你的概念**：

- Git 是啥、`add` / `commit` / `push` / `pull` 这 4 个命令。
- GitHub Pages 是啥、怎么开。
- VS Code 怎么打开文件夹、装 Live Server 插件做本地预览。

## 阶段 1 · 最小可玩原型（预计 2 ~ 3 天）

**目标**：一个**只有一关、丑得不行但能玩**的版本。

**做完的标志**：

- [ ] 屏幕上能看到一堆方块（用 emoji 或纯色 + 文字代替图标）。
- [ ] 方块有"被压住不能点"的判定（被压的发灰）。
- [ ] 能点击未被压住的方块，方块飞到底部槽。
- [ ] 槽里凑齐 3 个一样的会消掉。
- [ ] 槽满 7 个 → 弹"失败"。
- [ ] 牌全消完 → 弹"胜利"。
- [ ] 一个"重开"按钮。

**这阶段不做**：

关卡多样化、动画、音效、撤销 / 洗牌、皮肤、移动端适配。

**这阶段会写的文件**：

- `index.html`、`styles/base.css`、`src/main.js`、`src/state.js`、`src/config.js`
- `src/game/{tile, board, slot, rules}.js`
- `src/ui/{render, controls}.js`
- `levels/level-01.json`

---

## 阶段 2 · 视觉打磨（预计 2 天）

**目标**：变得不那么丑，玩起来"有手感"。

**做完的标志**：

- [ ] 用真实图标代替 emoji（建议先用 [Lucide](https://lucide.dev/) 或 [Twemoji](https://twemoji.twitter.com/) 这类开源图标）。
- [ ] 牌点击有放大 / 弹起动画（CSS transform）。
- [ ] 牌移到槽里有飞行动画。
- [ ] 三消有"消失"特效（缩放 + 淡出）。
- [ ] 失败 / 胜利有弹窗，不是 `alert()`。
- [ ] 加 1 ~ 2 个音效：点击声、消除声。

**这阶段会新增的文件**：

- `src/ui/effects.js`、`src/audio/sound.js`
- `assets/images/`、`assets/sounds/`
- `styles/theme.css`

---

## 阶段 3 · 关卡系统（预计 2 天）

**目标**：能玩多关，难度递增，进度能保存。

**做完的标志**：

- [ ] 至少 3 关（第一关教学，第二关常规，第三关变态）。
- [ ] 关卡选择界面。
- [ ] 通关一关解锁下一关。
- [ ] 玩家进度存进 `localStorage`，下次打开还在。
- [ ] 关卡数据全在 `levels/*.json` 里，加新关只加 JSON。

**这阶段会新增的文件**：

- `src/storage/localStorage.js`
- `src/game/level.js` 完善
- `levels/level-02.json`、`levels/level-03.json`

---

## 阶段 4 · 道具系统（预计 2 天）

**目标**：加经典三件套：撤销、移出、洗牌。

**做完的标志**：

- [ ] **撤销**：把上一步从槽里飞回去。
- [ ] **移出**：把槽里的 3 张牌放回牌堆顶。
- [ ] **洗牌**：把牌堆里所有未消除的牌打乱重排。
- [ ] 每个道具有使用次数限制（比如每关 3 次）。
- [ ] 道具用完按钮变灰。

**这阶段会新增的文件**：

- `src/game/powerups.js`（新文件）
- `src/state.js` 加 `history` 撤销栈

---

## 阶段 5 · 移动端适配 + 性能优化（预计 1 ~ 2 天）

**目标**：手机上也能流畅玩。

**做完的标志**：

- [ ] iPhone SE（320px 宽）上不错位。
- [ ] 手机上点击区域够大（≥ 40px）。
- [ ] 不出现 300ms 点击延迟。
- [ ] 在 Chrome DevTools 的 Performance 里跑一关，无明显卡顿。
- [ ] 首屏加载 ≤ 2 秒（本地测）。

---

## 阶段 6 · 上线 + 完善文档（预计 0.5 天）

**目标**：正式发布给朋友玩。

**做完的标志**：

- [ ] GitHub Pages 配好（可选自定义域名）。
- [ ] `README.md` 写完整：截图、玩法、技术栈、致谢。
- [ ] 加 LICENSE（建议 MIT）。
- [ ] 加一段"非商用、致敬原作"的声明。
- [ ] 发链接给朋友测一轮，收 bug 列表。

---

## 阶段 7+ · 可选高级功能（不限时间）

按你兴趣随便挑：

- **皮肤系统**：可切换牌面图案（猫主题、水果主题）。
- **每日挑战**：每天一关，全网同种子。
- **本地排行榜**：localStorage 存通关时间排序。
- **在线排行榜**：上 Firebase / Supabase。
- **关卡编辑器**：可视化拖拽生成关卡 JSON。
- **PWA**：离线可玩、可加到主屏幕。
- **国际化**：中英文切换。
- **AI 解题器**：自动玩通一关（学算法用）。

---

## 如何衡量"做完"

每阶段做完前，问自己 3 个问题：

1. 我能不能找个朋友说"你来玩玩"？
2. 关掉浏览器再开，状态还对吗？
3. 这阶段的代码 push 上 GitHub 了吗？

3 个都能回答"是"，才算真做完。
