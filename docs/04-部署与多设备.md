# 04 · 部署与多设备访问

> 怎么让朋友 / 你自己的手机 / 平板 都能玩到这个游戏。

有两条路：

| 方式 | 适合 | 麻烦度 | 受众 |
|---|---|---|---|
| **A. 局域网（dev-server）** | 现在立刻让自己的手机玩 | ⭐ 一键 | 同 WiFi 设备 |
| **B. GitHub Pages 公网部署** | 给朋友玩 / 多设备永久访问 | ⭐⭐ 一次配置后自动 | 全世界任何浏览器 |

两条路不冲突，建议都做。

---

## A. 局域网（手机连同 WiFi 直接玩）

### A.1 启动 dev-server

```powershell
npm start
```

启动后控制台会打印：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ▶ 咩了个咩 · 本地开发服务器
  ▶ 本机访问     : http://localhost:5500
  ▶ 同 WiFi 手机 : http://192.168.x.x:5500
  ▶ 按 Ctrl+C 停止
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### A.2 手机连同一个 WiFi

确保**电脑和手机在同一路由器下**，然后用手机浏览器打开上面那个 `http://192.168.x.x:5500`。

### A.3 Windows 防火墙提示

**第一次启动时**，Windows 可能会弹一个"是否允许 Node.js 访问网络"的窗口：

- 勾选**"专用网络"**（家里 WiFi）
- 点**"允许访问"**

如果错过没勾选，运行下面命令手动开放（**需要管理员 PowerShell**）：

```powershell
netsh advfirewall firewall add rule name="Sheep Game Dev Server" dir=in action=allow protocol=TCP localport=5500
```

### A.4 局限

- **必须电脑开着 + dev-server 跑着**才能访问
- 出门换网络（4G / 别人家 WiFi）就不行了
- 这只是临时方案，不能给朋友用

## B. GitHub Pages 公网部署（永久 + 全球访问）

### B.1 第一次配置（只做一次）

#### B.1.1 推送代码

```powershell
git push origin main
git push --tags
```

如果 push 卡住（常见网络问题），稍后再试，或者换个网络。

#### B.1.2 在 GitHub 仓库开启 Pages

1. 浏览器打开 <https://github.com/moyuyumoyuyumo/sheep-n-sheep>
2. 点顶部 **Settings**（仓库设置，不是账号设置）
3. 左侧菜单选 **Pages**
4. **Source** 选：**Deploy from a branch**
5. **Branch** 选：`main` / `/ (root)`，点 **Save**
6. 等 1～2 分钟，页面顶部会出现：
   > Your site is live at <https://moyuyumoyuyumo.github.io/sheep-n-sheep/>

### B.2 之后每次更新

每次本地改完 + commit + push，GitHub Pages 会自动重新部署（约 1～2 分钟）：

```powershell
git add -A
git commit -m "feat: 加新关卡"
git push origin main
```

### B.3 多设备访问

一旦 Pages 开起来，下面所有设备都能直接玩：

| 设备 | 怎么访问 |
|---|---|
| 自己的手机 | 浏览器打开 <https://moyuyumoyuyumo.github.io/sheep-n-sheep/> |
| 朋友的电脑 | 同上，或扫二维码 |
| 平板 | 同上 |
| 别人家电脑 / 公司电脑 | 同上 |

### B.4 给朋友分享 → 二维码

给手机访问的最快方式：用任何"网址转二维码"工具（如 <https://cli.im>）把 Pages URL 转成二维码，朋友扫一下就能玩。

### B.5 注意事项

- **localStorage 是设备隔离的**：在 A 设备攒的道具 / 通关进度 / 签到记录，B 设备看不到。这个项目不做账号系统。
- **GitHub Pages 是 HTTPS**：Web Audio / localStorage 都正常工作。
- **如果改了 GitHub 用户名 / 仓库名**：Pages URL 会变，需要重新生效。

## C. 隐藏功能 / 调试入口

下面这些在本地 / Pages 都生效。

### C.1 后门：满道具

无穷道具触发方式（任选）：

1. **键盘输 Konami code**：依次敲 `↑ ↑ ↓ ↓ ← → ← → B A`
2. **浏览器控制台敲**：`cheat()`

触发后右上角弹"🎰 道具拉满！"，三种道具立刻变 999。

### C.2 控制台调试 API

打开浏览器 F12 控制台，可以用：

```js
state                       // 当前游戏状态
startLevel(allLevels[5])    // 直接跳到任意关卡（注：allLevels 没暴露，要从 import 拿）
showMenu()                  // 回选关菜单
cheat()                     // 满道具
```

### C.3 清除存档（重置进度）

```js
localStorage.removeItem('mok-progress')   // 通关进度
localStorage.removeItem('mok-wallet')     // 道具余额
localStorage.removeItem('mok-daily')      // 今日签到记录
localStorage.removeItem('mok-muted')      // 静音偏好
location.reload()
```

## E. FAQ

**Q：手机访问 `192.168.x.x:5500` 拒绝连接？**
- 检查 Windows 防火墙是否放行 5500（A.3）
- 检查电脑和手机是否同一 WiFi（不是手机 4G）
- 重启一下 dev-server

**Q：Pages 部署失败 / 404？**
- 检查 Settings → Pages → Build and deployment 里 Branch 选了 `main`
- 检查 `index.html` 在仓库根目录（不在 `src/`）
- 等 2～3 分钟，第一次部署有延迟

**Q：手机上玩着玩着进度丢了？**
- 浏览器隐私模式 / 清除浏览器数据 会清掉 localStorage
- 不同浏览器（如 Chrome / Edge）也是隔离的
